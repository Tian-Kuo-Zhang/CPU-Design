# CPU 加解密系统说明书（EES331）

## 1. 系统目标
本系统在 EES331 开发板上实现基于自定义 CPU 指令集的 8 位加密/解密装置：
- 输入：`SW[7:0]`
- 输出：`LED[7:0]`
- 模式切换：`mode=0` 加密（`En`），`mode=1` 解密（`dE`）
- 数码管显示：`En/dE`

## 2. 本次修复背景
旧版程序在低 8 位输出场景下可能出现“不同输入对应相同密文”的现象，导致解密不稳定。

核心原因：
- 使用了面向 16 位寄存器的位旋转结果参与流程，但最终只显示低 8 位，可能破坏 8 位域上的一一映射。

修复策略：
- 改为严格在 8 位域上可逆的“查表置换链 + 模加减 + XOR”。
- 使用 involution 查找表（`LUT[LUT[x]] = x`），确保逆向步骤可直接复用同一张 LUT。

## 3. 新版加解密算法（无碰撞映射）
设输入 `x` 为 8 位值，常量：
- `K1 = 0x3D`
- `K2 = 0x6B`
- `K3 = 0x27`

查找表：`LUT`，满足 `LUT[LUT[v]] = v`。

### 3.1 加密（mode=0）
`Enc(x)`：
1. `x = x + K1`
2. `x = LUT[x]`
3. `x = x xor K2`
4. `x = x + K3`
5. `x = LUT[x]`

### 3.2 解密（mode=1）
`Dec(y)`：
1. `y = LUT[y]`
2. `y = y - K3`
3. `y = y xor K2`
4. `y = LUT[y]`
5. `y = y - K1`

因为每一步都是 8 位可逆变换，整体也是可逆变换，因此不会出现“一密多明”的碰撞。

## 4. 查表实现方式（load/store）
### 4.1 LUT 初始化
- `CPU_Top.v` 在上电时执行：
  - `$readmemh("crypto_lut.mem", data_ram);`
- `crypto_lut.mem` 预加载到 `data_ram[0..255]`。

### 4.2 程序查表动作
程序中通过：
- `load R1, #0x00`
实现 `R1 <- data_ram[R1[7:0]]`，即 `R1 <- LUT[R1]`。

说明：
- 查表使用 `load`；
- 输出使用 `store R1, #0x11` 写入 LED 映射地址。

## 5. 内存映射
- `0x0010`：`SW[7:0]`（只读）
- `0x0011`：`LED[7:0]`（写）
- `0x0012`：`mode`（bit0）

## 6. 程序文件
- 汇编注释：`program_crypto.asm`
- ROM 机器码：`program_crypto.mem`
- LUT 表：`crypto_lut.mem`

## 7. 使用说明
1. 在 Vivado 中将 `program_crypto.mem` 和 `crypto_lut.mem` 都加入工程。
2. 重新综合、实现并生成 bitstream。
3. 下载到开发板后：
   - `En` 模式输入明文得到密文；
   - 切到 `dE` 模式输入该密文，应恢复原明文。

## 8. 验证建议
- 随机取若干输入做 En->dE 往返测试，结果应恒等恢复。
- 如需穷举验证，可离线验证 0~255 全空间是否为一一映射。
