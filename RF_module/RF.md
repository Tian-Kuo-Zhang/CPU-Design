# Datapath Module Interface Specification: Register File (RF)

## 1. Module Overview
* **Module Name:** `RF`
* **Description:** A 4-word × 16-bit multi-port Register File. It features a dual-read, single-write architecture designed to supply two source operands simultaneously to the ALU and store one result per clock cycle.
* **Key Feature:** Register 0 ($R_0$) is hardwired to a constant `16'b0` at the hardware level to support zero-value generation and ensure architectural consistency.

## 2. Interface Definition (Port Map)
The module communicates with the instruction decoder and the broader datapath via the following ports:

| Port Name | Direction | Width (bits) | Description |
| :--- | :--- | :--- | :--- |
| `clk` | Input | 1 | System clock signal. Write operations are synchronized to the rising edge. |
| `RegWrite` | Input | 1 | Write Enable control signal generated by the OC. When asserted (`1`), it allows data to be written into the specified register. |
| `raddr1` | Input | 2 | Read Address 1. Corresponds to the `Rs` field in the instruction. |
| `raddr2` | Input | 2 | Read Address 2. Corresponds to the `Rt` field in the instruction. |
| `waddr` | Input | 2 | Write Address. Corresponds to the `Rd` field in the instruction. |
| `wdata` | Input | 16 | Write Data bus. Carries the 16-bit result from the ALU or Data Memory. |
| `rdata1` | Output | 16 | Read Data 1 bus. Outputs the 16-bit value stored in the register pointed to by `raddr1`. |
| `rdata2` | Output | 16 | Read Data 2 bus. Outputs the 16-bit value stored in the register pointed to by `raddr2`. |

## 3. Functional Characteristics
* **Asynchronous Read:** The read ports (`rdata1` and `rdata2`) operate asynchronously. Any change in `raddr1` or `raddr2` immediately reflects the updated register contents on the output buses within the same clock cycle.
* **Synchronous Write:** The write operation is strictly synchronous. Data present on the `wdata` bus is latched into the register specified by `waddr` only on the **rising edge** of `clk`, provided that `RegWrite == 1`.
* **Zero-Register Protection:** Any read access to address `2'b00` ($R_0$) will strictly output `16'h0000`. Any write attempt to address `2'b00` is explicitly ignored by the internal logic, safeguarding the constant zero state.
