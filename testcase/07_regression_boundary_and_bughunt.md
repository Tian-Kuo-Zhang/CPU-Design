# 07 Regression/Boundary - Bug Hunt

## 目标

把前面测试整合成回归集，重点攻击边界行为。

## 回归集合

1. `R0` 防写保护：多次尝试写 `R0`，确认始终为 0。
2. ALU 溢出回绕：`0xFFFF + 2 -> 0x0001`。
3. RAM 地址别名：访问 `0x0001` 与 `0x0101`，确认是否被映射到同一单元（当前实现只用低 8 位）。
4. 分支边界：`PC=0xFE` 时再跳转，观察是否越界回绕。
5. 随机指令流（20~50 条）与软件模型比对（寄存器和 RAM 终态）。

## 建议自动判分项

- 每条程序结束时导出：`PC, R0~R3, RAM[关键地址]`。
- 与 Python 参考模型逐项比对，输出 PASS/FAIL。

## 当前代码中最可能出现的 CPU bug（按优先级）

1. **字段重叠设计问题（严重）**
   - `rt` 与 `imm[7:6]` 复用同一比特，导致 `store/beq/bne` 无法完整表达“两个寄存器 + 8 位立即数”。
2. **分支负偏移问题（严重）**
   - `FetchUnit` 对分支偏移未做符号处理，负跳转可能不符合预期。
3. **RAM 地址空间截断（中）**
   - 仅使用 `addr[7:0]`，高位地址冲突。
4. **未初始化 RAM 导致 X 态传播（中）**
   - 首次读未写地址可能出现不确定值，影响分支和写回观测。

## 你下一步可以直接做的修复方向

- 修正 ISA 编码（至少让 `store/beq/bne` 的寄存器字段与立即数字段不重叠）。
- 明确分支偏移是否有符号；若是有符号，Fetch 端改为符号扩展后相加。
- 给 RAM 增加仿真初始化，降低 X 态噪声。
