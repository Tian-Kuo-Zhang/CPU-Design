# 05 Integration Test - Memory Path

## 目标

验证 `RF -> ALU(地址) -> RAM -> WriteBack` 这条链路，并主动检查编码边界问题。

## 测试程序 A（基础通路）

1. `1404` ; mov R1, #4      (base)
2. `1809` ; mov R2, #9      (data)
3. `3100` ; store R0?, #0   (这里只能安全用 rt=00，见下面 bug 说明)
4. `2480` ; load R1, #0x80  (负偏移读，检查符号扩展路径)

> 程序 A 主要是让 RAM 链路可观测；真实 `store R2, imm` 会受到编码冲突限制。

## 测试程序 B（专门抓编码 bug）

目标指令语义：`store R2, #1`（基址假设在 R1）

如果按当前字段连线：

- `rt` 来自 `instr[7:6]`
- `imm` 来自 `instr[7:0]`

则 `rt` 和 `imm[7:6]` 重叠，无法独立指定。

## 预期结果

- A 程序可跑通 RAM 写读基本路径（至少在受限编码下可观测到变化）。
- B 测试应暴露“无法同时自由选择 `rt` 与 8 位 `imm`”的问题。

## 能抓到的 bug（高优先级）

- **编码冲突 bug**：`store/beq/bne` 的寄存器字段与 `imm[7:6]` 重叠，和文档中的“完整 8 位立即数”不一致。
- 地址符号扩展 bug：负偏移若行为异常，通常是符号扩展或地址计算链路问题。
