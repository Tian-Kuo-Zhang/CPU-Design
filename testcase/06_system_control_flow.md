# 06 System Test - Control Flow

## 目标

系统级验证 `beq/bne/jump` 与 PC 更新协同。

## 测试程序（建议拆两轮）

### Round 1: 正向跳转

1. `1403` ; mov R1, #3
2. `1803` ; mov R2, #3
3. `7102` ; beq R1, R0?, +2  (用于观察分支生效，具体寄存器按你当前编码表换算)
4. `1CFF` ; mov R3, #0xFF     (陷阱指令，理论上应被跳过)
5. `6007` ; jump #7
6. `1400` ; mov R1, #0         (陷阱指令，理论上应被跳过)
7. `1801` ; mov R2, #1         (落点)

### Round 2: 负偏移回跳（关键）

构造 `bne` 回跳形成小循环（offset 期望为负值，例如 `0xFE` 表示 -2）。

## 预期结果

- Round 1：陷阱指令不执行，PC 命中目标地址。
- Round 2：若支持有符号分支偏移，应形成循环计数行为。

## 能抓到的 bug（高价值）

- **FetchUnit 分支立即数未符号扩展**：当前实现 `PC_branch = PC_out + imm_8`，`imm_8` 是原始 8 位无符号，负跳转很可能失败。
- **分支比较寄存器选择异常**：若 `beq/bne` 比较对象不对，通常是字段映射问题。
