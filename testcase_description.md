### 🌀 什么是“组合逻辑环路 (Combinational Loop)”？

在数字电路中，**组合逻辑**（比如加法器、多路选择器、导线）是没有记忆功能的，信号一端输入，另一端瞬间（忽略物理延迟）输出。
如果你把一个组合逻辑的输出，**没有经过任何触发器（Flip-Flop，即没有时钟沿控制）**，直接又连回了它的输入端，这就形成了一个**死循环**。

举个最简单的例子：你写了一句 `assign A = A + 1;`。
在真实物理世界里，这会导致电路疯狂震荡发热甚至烧毁；在 Vivado 仿真器里，由于信号在这条环路里转一圈不需要任何时间（0 纳秒），仿真器会在同一时刻（比如 `35ns` 这一瞬间）疯狂计算 10000 次，然后崩溃报错：`Zero Delay Oscillation`。

### 🔙 案情重演：我们原来是怎么写的？（触发 Bug 的代码）

为了解决所谓“读到旧数据”的时序问题，我当时让你在 `RF_module.v`（寄存器堆）里加了一段**“内部转发”**的组合逻辑。

当时**错误（触发死锁）的代码**是这样的：

```verilog
// 【引发死循环的代码】带有内部转发的异步读取
assign rdata1 = (raddr1 == 2'b00) ? 16'b0 : 
                ((RegWrite && waddr == raddr1) ? wdata : registers[raddr1]);
```

**这段代码的初衷是：** 当你正在对某一个寄存器执行写操作（`RegWrite == 1`），且写的地址和读的地址一样时（`waddr == raddr1`），不要去读数组里旧的值了，**直接抄近道**，把正在写入的 `wdata` 送给读端口 `rdata1`。

---

### 💥 灾难是如何爆发的？（环路的形成）

在流水线（Pipeline）CPU 中，上面那种“转发”写法是救命的。但在你的**单周期（Single-Cycle）CPU** 中，它是致命的！

当你的 CPU 读到正确的指令 **`subi R1, R1, 3` (机器码 5503)** 时，灾难发生了：

1. **指令意图：** `R1 = R1 - 3`。这要求 CPU 既要读 `R1`，又要写 `R1`。
2. **读写地址匹配：** 此时 `raddr1` 是 `1`，`waddr` 也是 `1`。`RegWrite` 为 `1`。
3. **闭环形成，开始“套娃”：**
   * ALU 想要计算减法，向寄存器堆要 `rdata1`。
   * 寄存器堆看到 `waddr == raddr1`，触发了上面的转发逻辑：**“你要 `rdata1` 是吧？我把 `wdata` 直接给你！”**
   * 但是，`wdata` 是哪里来的？在 CPU 的数据通路连线中，`wdata` 连着 ALU 的计算结果 `ALU_result`。
   * ALU 喊道：**“你把 `wdata` 给我没用啊，我的结果还没算出来呢！我得先拿到 `rdata1` 才能算出 `wdata` 啊！”**

**死锁的逻辑等式：**
`rdata1` = `wdata` 
`wdata` = `ALU_result`
`ALU_result` = `rdata1` - 3

综合起来就是：**`rdata1` = `rdata1` - 3**。

这是一个没有任何时钟（D触发器）阻隔的纯组合逻辑环路！电信号像衔尾蛇一样疯狂自我吞噬，Vivado 算不出结果，只能强行拉闸报错。

---

### 🛠️ 我们后来是怎么改的？（解开环路的正确代码）

怎么打破这个死循环？非常简单：**斩断这条“抄近道”的组合逻辑连线，把时钟（Clock）这堵墙重新竖起来！**

我们去掉了那段花哨的判定，**改回了你最初写的代码**：

```verilog
// 【正确的代码】纯粹的异步读取
assign rdata1 = (raddr1 == 2'b00) ? 16'b0 : registers[raddr1];

// 同步写入保留在时钟上升沿
always @(posedge clk) begin
    // ...
end
```

**为什么改回这样就对了？**
因为单周期 CPU 的核心哲学就是**“时间换空间”**：
1. **组合逻辑阶段（半个周期）：** ALU 老老实实从 `registers[1]` 这个物理存储器里读出旧值（比如 `0x47`），算出 `0x44`。此时 `0x44` 停留在 `wdata` 的门口。
2. **时钟上升沿（瞬间）：** 时钟 `clk` 敲响，`0x44` 正式被写入 `registers[1]`。
3. **物理隔离：** 读（组合逻辑）和写（时序逻辑）被时钟沿严格隔开，`rdata1` 永远不会直接等于 `wdata`，自然就不会有组合逻辑环路了。

这段排错经历非常经典。你可以直接在报告里画一个流程图，把 `rdata1 -> ALU -> wdata -> rdata1` 这个死循环画出来，这绝对是这门实验课上最闪光的“加分项”！还需要我帮你构思这段内容的图文排版建议吗？
